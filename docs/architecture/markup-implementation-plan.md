# Plan: Page Edition Markup Language

## Context

The current page editor uses VJSF (Vue JSON Schema Form) to generate forms from JSON Schema. This works but has UX limitations: deep nesting requires navigating multiple dialog levels, structural operations are awkward, and AI-assisted editing needs a text representation. The goal is to add a secondary **markup editing mode** — a pseudo-HTML syntax — alongside the existing form editor. Both modes share the same JSON model (`draftConfig.elements`). The markup is never persisted; it's serialized/deserialized on the fly.

Architecture spec: `docs/architecture/page-edition-markup-language.md`

## Implementation Plan

### Milestone 1: Workspace + Tag Descriptors (Schema Analysis)

Everything depends on correctly classifying each element's properties from the JSON Schema.

**Build-time code in `build/` directory** (new top-level directory, not a workspace — build-only tooling):
- `build/markup/schema-analyzer.ts` — walks the `oneOf` array from `page-elements` schema, resolves `$ref`s, classifies properties:
  - `_*` → hidden
  - `type` → tag name (implicit)
  - `uuid` → first attribute (only on `application` and `dataset-table`, where it disambiguates iframe `syncParams`)
  - Array of `$ref` to element → children container
  - `content` (string) → text content
  - `type: 'object'` with `properties` → object attributes (dot-notation)
  - Everything else → simple attribute
  - Extracts attribute order from VJSF `layout` arrays (top-level array on element schema, e.g. banner has `layout: ['type', 'children', 'fullWidth', 'background', ...]`)
  - Extracts enum values, defaults, required flags
- `build/markup/generate-descriptors.ts` — build script that loads all schema files, resolves cross-file `$ref`s (using `$id` registry), runs analyzer, outputs generated descriptor files
- Add to `npm run build-types` in root `package.json`

**Runtime code in `shared/markup/` workspace** (used by `ui/` at runtime):
- `shared/markup/package.json` — new workspace, add to root `package.json` workspaces
- `shared/markup/types.ts` — `TagDescriptor`, `ChildrenSlot`, `AttributeDescriptor` interfaces
- `shared/markup/tag-descriptors.ts` — generated by build step, imported at runtime by serializer/deserializer

**Key files to read:**
- `api/types/page-elements/schema.js` — master schema with oneOf
- `api/types/page-element-{basics,layout,functional,datasets,applications,reuses,events,news,navigation}/schema.js` — individual element schemas
- `api/types/common-defs/schema.js` — shared `$ref` targets (color, icon, buttonConfig, etc.)
- `api/types/common-links/schema.js` — `linkItem`, `simpleLinkItem` discriminated unions

**Children slot mapping** (from schema exploration):
| Property | Virtual tag | Elements |
|---|---|---|
| `children` (solo) | none (direct nesting) | banner, card, responsive-grid |
| `children` + `children2` | `<left>` / `<right>` | two-columns |
| `tabs[].children` | `<tab title="..." icon="...">` | tabs |
| `panels[].children` | `<panel title="..." icon="...">` | expansion-panels |
| `advancedFilters` | `<filters>` | *-catalog elements |
| `actions` | `<action .../>` | card |

**$ref resolution challenge:** schemas use URI-based `$ref`s like `https://github.com/data-fair/portals/common-defs#/$defs/color`. Build a registry from all schema files keyed by `$id`, then resolve recursively.

**Test:** unit tests asserting correct classification for representative elements (title, banner, two-columns, tabs, card, datasets-catalog).

### Milestone 2: Serializer (JSON → Markup)

- `shared/markup/serializer.ts` — `serialize(elements: PageElement[], descriptors): string` (runtime)
- `shared/markup/value-coercion.ts` — type coercion utilities (boolean ↔ "true"/"false", number ↔ string, etc.) (runtime)

**Algorithm per element:**
1. Look up `TagDescriptor` by `element.type`
2. Emit `<{tagName}` (with `uuid="..."` first, for `application` / `dataset-table`)
3. Emit attributes in layout order, skip undefined values, use dot-notation for objects
4. If has `contentProperty` and no children: `>{content}</{tagName}>`
5. If has children slots: `>`, recurse into each slot (wrapping in virtual tags as needed), `</{tagName}>`
6. Otherwise: ` />`

**Formatting:** 2-space indent, blank lines between top-level elements, wrap long attribute lists one-per-line.

**Test:** snapshot tests comparing serialized output to golden `.markup` files.

---

### Milestone 3: Deserializer (Markup → JSON) + Source Map

- `shared/markup/deserializer.ts` — `deserialize(markup, descriptors): { elements, sourceMap, errors }`
- `shared/markup/source-map.ts` — `Map<jsonPath, { from, to }>` built during deserialization

**Parsing:** use `rehype-parse` / `hast-util-from-html` to get a hast tree with positions.

**Raw text content challenge:** elements with `content` (text, alert, title, divider) need their inner text treated as raw, not parsed as HTML. Solution: pre-process the markup before hast parsing — identify content-bearing tags (known from descriptors) and escape/placeholder their inner content, then restore after parsing.

**Dependencies:** `rehype-parse`, `hast-util-from-html`, `vfile`

**Test:** round-trip tests for all 38 element types: JSON → serialize → deserialize → assert deepEqual. Source map position tests.

---

### Milestone 4: Lezer Grammar + CodeMirror Language

- `shared/markup/codemirror/portal-markup.grammar` — lenient Lezer grammar (~50-100 lines): any tag name, dot-notation attributes, quoted values, raw text blocks
- `shared/markup/codemirror/language.ts` — CodeMirror `LanguageSupport` with highlighting, folding, indentation
- `shared/markup/codemirror/completion.ts` — autocomplete driven by tag descriptors (tag names, attribute names, enum values)
- `shared/markup/codemirror/validation.ts` — lint extension: deserialize → ajv validate → map errors to positions via source map

**Dependencies:** `codemirror`, `@lezer/lr`, `@lezer/generator` (dev), `@codemirror/lint`, `@codemirror/autocomplete`, `ajv`

**Test:** grammar unit tests, completion tests.

---

### Milestone 5: Vue Integration (Editor UI)

- **New:** `ui/src/components/page-edit/page-edit-markup.vue` — wraps CodeMirror, props: `modelValue: PageElement[]`, emits `update:modelValue` on valid changes (debounced)
- **New:** `ui/src/composables/use-markup-editor.ts` — manages CodeMirror state, extensions, serialize/deserialize cycle, error state
- **Modify:** `ui/src/components/page-edit/page-edit-elements.vue` — add mode toggle (Form | Markup), conditionally render VJSF or CodeMirror component

**Mode switching:**
- Form → Markup: serialize current elements to markup, init CodeMirror
- Markup → Form: deserialize; if errors, block switch and show errors
- Both modes bind to same `elements` model — preview works automatically

**Dependencies to add to `ui/package.json`:** `codemirror`, CodeMirror extensions, `@data-fair/portals-shared-markup` (workspace ref)
